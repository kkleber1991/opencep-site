# Contributing to OpenCEP Ultra

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to **OpenCEP Ultra**. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue and include:
- What happened.
- What you expected to happen.
- Steps to reproduce.
- Your OS and environment (if relevant).

### Suggesting Enhancements
We love suggestions! Please describe:
- The specific feature you'd like to see.
- Why it would be useful.
- How it should work (if you have an idea).

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  If you've changed APIs, update the documentation.
4.  Ensure the test suite passes.
5.  Issue that pull request!

## Style Guide
- Maintain the **clean and minimalist** aesthetic of the landing page.
- For JSON data structures, use the **standard OpenCEP** format to ensure backward compatibility.

## Attribution
By contributing, you agree that your contributions will be licensed under the project's **MIT License**.
